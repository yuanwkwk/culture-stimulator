# 属性影响和触发条件设置指南

## 功能概述

在创作文化模板时，用户现在可以通过图形化界面设置：
1. **事件的触发条件**：决定事件何时出现
2. **选择项的属性影响**：决定选择后对属性的影响
3. **结局的触发条件**：决定结局何时触发

## 一、事件触发条件

### 什么是触发条件？
触发条件决定了事件在游戏中何时出现。只有当玩家的属性满足所有设置的条件时，该事件才会触发。

### 如何设置？
1. 在"人生事件"标签页中创建或编辑事件
2. 找到"触发条件（可选）"部分
3. 为每个属性设置最低要求值（0-100）
4. 留空表示该属性无要求

### 示例

#### 示例1：科举考试
```
事件：科举考试
触发条件：
  - 学识 ≥ 50
  - 声望 ≥ 30
```
**含义**：只有当玩家的学识达到50以上，且声望达到30以上时，才会触发科举考试事件。

#### 示例2：商业机遇
```
事件：重大商业机遇
触发条件：
  - 财富 ≥ 60
  - 商业头脑 ≥ 70
  - 人脉关系 ≥ 50
```
**含义**：需要同时满足财富、商业头脑和人脉关系的要求。

#### 示例3：无条件事件
```
事件：家庭背景
触发条件：（全部留空）
```
**含义**：无条件触发，在该人生阶段随机出现。

### 设置建议

#### 童年期事件
- 建议设置较低的条件或无条件
- 例如：学识 ≥ 10，健康 ≥ 20

#### 青少年期事件
- 可以设置中等条件
- 例如：学识 ≥ 30，技艺 ≥ 25

#### 成年期事件
- 可以设置较高条件
- 例如：财富 ≥ 50，声望 ≥ 40

#### 晚年期事件
- 可以设置很高的条件或特殊组合
- 例如：学识 ≥ 70，声望 ≥ 60

## 二、选择项属性影响

### 什么是属性影响？
属性影响决定了玩家选择某个选项后，各项属性会如何变化。

### 如何设置？
1. 在事件的选择项中找到"属性影响"部分
2. 为每个属性设置影响值
3. 正数表示增加，负数表示减少
4. 留空或0表示无影响

### 示例

#### 示例1：科举考试的选择
```
选择1：努力学习，准备考试
属性影响：
  - 学识 +15
  - 健康 -5
  - 声望 +10
```
**含义**：选择努力学习会增加15点学识和10点声望，但会减少5点健康。

```
选择2：放弃考试，专心经商
属性影响：
  - 财富 +10
  - 商业头脑 +15
  - 声望 -5
```
**含义**：选择经商会增加财富和商业头脑，但会减少声望。

#### 示例2：健康选择
```
选择1：坚持锻炼
属性影响：
  - 健康 +20
  - 技艺 +5
```

```
选择2：忽视健康
属性影响：
  - 健康 -10
  - 财富 +15
```

### 设置建议

#### 平衡性原则
- 有得必有失：增加某些属性时，适当减少其他属性
- 例如：学习增加学识但减少健康

#### 数值范围
- 小影响：±5
- 中等影响：±10-15
- 大影响：±20-30
- 避免单次影响超过±50

#### 逻辑性
- 确保影响符合现实逻辑
- 例如：努力学习增加学识，而不是增加财富

#### 多样性
- 不同选择应该有明显不同的影响
- 避免所有选择都只影响一两个属性

## 三、结局触发条件

### 什么是结局触发条件？
结局触发条件决定了玩家在游戏结束时会获得哪个结局。系统会根据玩家的最终属性，匹配满足条件的结局。

### 如何设置？
1. 在"人生结局"标签页中创建或编辑结局
2. 找到"触发条件"部分
3. 为每个属性设置最低要求值（0-100）
4. **必须至少设置一个条件**

### 示例

#### 示例1：传奇结局
```
结局：诗仙传世
成就等级：传奇
触发条件：
  - 学识 ≥ 85
  - 声望 ≥ 80
  - 技艺 ≥ 75
```
**含义**：需要极高的学识、声望和技艺才能达成。

#### 示例2：卓越结局
```
结局：富甲一方
成就等级：卓越
触发条件：
  - 财富 ≥ 80
  - 商业头脑 ≥ 75
  - 人脉关系 ≥ 60
```
**含义**：专注商业发展的玩家可以达成。

#### 示例3：良好结局
```
结局：平凡幸福
成就等级：良好
触发条件：
  - 健康 ≥ 60
  - 财富 ≥ 40
```
**含义**：保持健康和基本财富即可达成。

#### 示例4：普通结局
```
结局：平淡一生
成就等级：普通
触发条件：
  - 学识 ≥ 30
```
**含义**：只需要基本的学识即可达成。

#### 示例5：困顿结局
```
结局：穷困潦倒
成就等级：困顿
触发条件：
  - 财富 ≤ 20
  - 健康 ≤ 30
```
**含义**：财富和健康都很低时触发（注意：当前系统只支持≥条件，此示例需要特殊处理）。

### 设置建议

#### 成就等级对应的条件范围

**传奇（5星）**
- 至少3-4个属性要求
- 属性值：≥ 80-90
- 例如：学识≥85, 声望≥80, 技艺≥75

**卓越（4星）**
- 至少2-3个属性要求
- 属性值：≥ 70-80
- 例如：财富≥75, 商业头脑≥70

**良好（3星）**
- 至少1-2个属性要求
- 属性值：≥ 55-70
- 例如：健康≥60, 财富≥55

**普通（2星）**
- 至少1个属性要求
- 属性值：≥ 30-50
- 例如：学识≥40

**困顿（1星）**
- 可以设置很低的要求或特殊条件
- 属性值：≥ 10-30
- 例如：健康≥15（作为兜底结局）

#### 结局覆盖策略
- 确保有兜底结局（条件很低的结局）
- 不同路线应该有对应的结局
- 避免条件重叠导致无法触发某些结局

#### 结局数量建议
- 最少：3个（传奇、良好、普通）
- 推荐：5-8个（覆盖不同路线和等级）
- 最多：10-15个（提供丰富的结局体验）

## 五、常见问题

### Q1: 如果不设置触发条件会怎样？
A: 事件会在该人生阶段随机出现，不受属性限制。建议童年期事件不设置条件，后期事件设置条件。

### Q2: 属性影响可以设置为0吗？
A: 可以。设置为0或留空都表示该属性不受影响。

### Q3: 如果多个结局的条件都满足怎么办？
A: 系统会选择条件最严格（要求最高）的结局。建议设计时避免条件重叠。

### Q4: 属性值的范围是多少？
A: 属性值范围是0-100。初始值通常在20-50之间，最终值可能达到80-100。

### Q5: 如何设计平衡的属性影响？
A: 遵循"有得必有失"原则。例如：
- 努力学习：学识+15，健康-5
- 拼命工作：财富+20，健康-10
- 休息养生：健康+15，财富-5

### Q6: 结局必须设置触发条件吗？
A: 是的，每个结局至少需要设置一个触发条件。这确保结局有明确的达成要求。

### Q7: 如何测试设置的条件是否合理？
A: 提交后可以在游戏中测试。建议：
1. 先创建简单的测试模板
2. 玩几次游戏验证
3. 根据体验调整条件

## 六、最佳实践

### 1. 渐进式难度
- 童年期：无条件或低条件
- 青少年期：中等条件
- 成年期：较高条件
- 晚年期：高条件或特殊组合

### 2. 多样化路线
- 设计不同的发展路线（学术、商业、艺术等）
- 每条路线有对应的事件和结局
- 避免单一路线过于强势

### 3. 平衡性考虑
- 属性影响不要过大
- 确保每个选择都有价值
- 避免"必选"选项

### 4. 逻辑一致性
- 属性影响符合现实逻辑
- 触发条件合理
- 结局与玩家的选择相符

### 5. 玩家体验
- 提供清晰的反馈
- 避免过于复杂的条件
- 确保有多种达成结局的方式
